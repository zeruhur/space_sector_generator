# System Detail Expansion

This chapter expands a generated SEP into a fully detailed star system. The SEP remains the canonical summary, but this layer turns that summary into planets, moons, habitats, stations, and local settlement structure.

## Design goal

The purpose of this expansion is to preserve the speed and clarity of the SEP while adding enough physical detail to make a system playable as a location. The SEP tells you what matters; this chapter tells you what is there.

A system generated by this chapter should always answer three questions:

- What does the system contain?
- Who uses it or lives in it?
- Why does it matter in play?


## Using this chapter

Generate the SEP first. Then use the SEP fields to shape the system's astronomical and infrastructural detail.

The procedure below assumes that the sector map and system profile already exist. This chapter adds detail to one system at a time.

The full procedure uses only two six-sided dice (2d6) and one six-sided die (1d6).

## System record

Every detailed system is recorded using this template:

```text
[Canonical Identifier]
[Local Name]
[SEP Profile]

Star Architecture:
- Number of stars.
- Star type and size.
- Special conditions.

Orbital Bodies:
- Planet list.
- Moon list.
- Belt presence.

Artificial Structures:
- Habitats.
- Stations.
- Bases.
- Colonies.

Population:
- Scale.
- Distribution.
- Major settlements.

Technology:
- Level.

Local Conditions:
- Access.
- Hazards.
- Tension.
- Distinctiveness.
- Travel and route notes.

Referee Notes:
- What matters here.
- What is dangerous here.
- What makes this system memorable.
```


## Step 0: SEP framing

Before rolling anything, read the SEP profile and note what each field implies. The generation steps below produce physical texture; the SEP defines the outcome. If a roll produces a result that contradicts the profile, the profile wins.

**Hz** establishes environmental baseline. High Hz means hostile physical conditions apply to most or all bodies in the system.

**Rx** establishes economic identity. The resource type determines what is worth exploiting and what traffic the system attracts.

**Pp** establishes population expectation. Use it to weight population rolls and decide how many bodies are plausibly inhabited.

**Pw** establishes political structure. Use it to assign control over stations, habitats, and colonies without re-rolling authority separately.

**Tn** establishes conflict context. High Tn means security checkpoints, contested facilities, blockades, or active structural damage.

**Dx** establishes the system's hook. If Dx is set, at least one physical feature should visibly express it.


## Step 1: Star architecture

Determine the system's stellar structure first. This establishes the physical frame for everything else.

### Number of stars

Roll 1d6.

| 1d6 | Result |
| :-- | :-- |
| 1-3 | Single |
| 4-5 | Binary |
| 6 | Multiple — roll 1d6: 1-4 Trinary, 5-6 Quaternary or more |

For binary and multiple systems, roll for each star separately. The largest body is the primary.

### Star spectral type

Roll 1d6.

| 1d6 | Type |
| :-- | :-- |
| 1-4 | M (red dwarf) |
| 5 | K (orange dwarf) |
| 6 | Roll 1d6 on the extended table |

**Extended table (1d6):**

| 1d6 | Type |
| :-- | :-- |
| 1-2 | G (yellow) |
| 3 | F (yellow-white) |
| 4 | White Dwarf |
| 5 | A (white) |
| 6 | Exotic — roll 1d6 on the exotic table |

**Exotic star table (1d6):**

| 1d6 | Type |
| :-- | :-- |
| 1-3 | Brown Dwarf |
| 4 | Red Giant |
| 5 | Blue Giant |
| 6 | Roll 1d6: 1-4 Neutron Star, 5-6 Black Hole |

### Star size

Do not roll for exotic stars or White Dwarfs.

Roll 1d6.

| 1d6 | Size |
| :-- | :-- |
| 1-5 | Dwarf |
| 6 | Roll 1d6: 1-3 Giant, 4-5 Supergiant, 6 Hypergiant |

### Star notes

Use the following prompts to add detail:

- Does the star configuration create stable habitable zones?
- Is the system visually distinctive?
- Does the stellar environment create access hazards?
- Are there unusual radiation, gravity, or navigation conditions?


## Step 2: Planetary inventory

Determine how many major orbital bodies the system contains.

### Number of planets

Roll 1d6+4.

### Planet type

Roll 1d6 per planet.

| 1d6 | Type |
| :-- | :-- |
| 1 | Gas Giant |
| 2-3 | Ice Giant |
| 4-5 | Terrestrial |
| 6 | Roll 1d6: 1-3 Dwarf Planet, 4-6 Asteroid Belt |

### Habitability and placement

Roll 1d6 to determine how many planets lie in the habitable zone.

| 1d6 | Worlds in habitable zone |
| :-- | :-- |
| 1-4 | One |
| 5 | Two |
| 6 | Three |

These need not all be Earthlike. They are the bodies most likely to support complex settlement.

### Terrestrial features

Roll 1d6 for terrain category, then roll 1d6 for the specific type within that category.

**Terrain category (1d6):**

| 1d6 | Category |
| :-- | :-- |
| 1 | Hostile |
| 2 | Cold |
| 3 | Dry |
| 4 | Temperate |
| 5 | Wet |
| 6 | Exceptional |

**Specific type (1d6 within category):**

| 1d6 | Hostile | Cold | Dry | Temperate | Wet | Exceptional |
| :-- | :-- | :-- | :-- | :-- | :-- | :-- |
| 1-2 | Barren | Frozen | Desert | Savanna | Tropical | Hothouse |
| 3-4 | Hellworld | Arctic | Arid | Continental | Archipelago | Garden World |
| 5 | Tainted | Glacial | Steppe | Forest | Waterworld | Ecumenopolis |
| 6 | Ruined | Ice Shell | Tidally Locked | Relic | Flooded | Anomalous |

Tidally Locked is an orbital condition and may overlay any other terrain type in the same category.

### Planet detail prompts

For each planet, record:

- Orbit position.
- Type.
- Surface or atmospheric feature.
- Habitability.
- Resource value.
- Population presence.
- Major installations.
- Notable hazards or anomalies.


## Step 3: Moon inventory

Determine moons for each natural body that can plausibly have them.

### Moon counts

Roll for each body separately.

**Gas Giants:** Roll 2d6 for the number of moons.

**Ice Giants:** Roll 1d6+1 for the number of moons.

**Terrestrial planets:** Roll 1d6.

| 1d6 | Moons |
| :-- | :-- |
| 1-3 | None |
| 4-5 | One |
| 6 | Two |

**Dwarf planets:** Roll 1d6. On a 6, the body has one satellite. Otherwise none.

**Asteroid belts and space habitats:** No moons.

### Moon size and type

Roll 1d6 per moon.

| 1d6 | Type |
| :-- | :-- |
| 1-3 | Small Rock |
| 4-5 | Big Rock |
| 6 | Roll 1d6: 1-4 Planetary-mass, 5-6 Ring system |

If the result is Planetary-mass, roll on the terrestrial features table to determine its type.

### Moon detail prompts

For each moon, note:

- Parent body.
- Type.
- Habitability.
- Resource role.
- Settlement presence.
- Strategic use.


## Step 4: Space habitats

Space habitats are major artificial living spaces, usually independent of planetary habitability.

### Number of habitats

Roll 1d6.

| 1d6 | Count |
| :-- | :-- |
| 1-4 | None |
| 5 | One |
| 6 | Roll 1d6: 1-4 two habitats, 5-6 three habitats |

### Habitat type

Roll 1d6 per habitat.

| 1d6 | Type |
| :-- | :-- |
| 1-2 | O'Neill Cylinder |
| 3 | Stanford Torus |
| 4 | Bernal Sphere |
| 5 | Bishop Ring or Banks Orbital |

**Dyson Sphere and Ringworld:** if either is generated, treat all planets and moons in the system as unsettled regardless of earlier rolls, unless the referee decides otherwise.

### Habitat detail prompts

For each habitat, determine:

- Type.
- Location.
- Owner.
- Population.
- Primary function.
- Structural condition.
- Connection to trade or defense.

## Step 7: Population layer

Population should be written as visible settlement texture, not as a bare number alone.

### Population scale

Roll 1d6 per inhabited body. Add the system's Pp value to the result.

**Modifier:** subtract 1 if the body carries a significant local hazard, or if Tn is 4 or higher.

| Modified total | Population |
| :-- | :-- |
| 1 or less | Uninhabited |
| 2 | Outpost (10+) |
| 3 | Foothold (100+) |
| 4 | Settlement (1,000+) |
| 5 | Town (10,000+) |
| 6 | City (100,000+) |
| 7 | Metropolis (1,000,000+) |
| 8 | Major World (10,000,000+) |
| 9 | Regional Center (100,000,000+) |
| 10 | Core World (1,000,000,000+) |
| 11+ | Ecumenopolis (10,000,000,000+) |

### Population interpretation

Use the scale to decide:

- How many settlements exist.
- Whether there are multiple urban centers.
- Whether the system feels frontier-like or metropolitan.
- Whether stations dominate over planets.
- Whether the population is concentrated or spread across many habitats.


## Step 8: Technology level

Roll 2d6. Add the system's Pp value.

| Modified total | Level |
| :-- | :-- |
| 5 or less | Pre-Industrial |
| 6-7 | Machine Age |
| 8-9 | Atomic Age |
| 10-11 | Information Age |
| 12-13 | Space Age |
| 14-15 | Stellar Age |
| 16+ | Interstellar Age |

Government type and economic character are drawn from the SEP Pw and Rx fields respectively. They do not require additional rolls.


## Step 9: SEP reconciliation

After completing Steps 1-8, review the SEP profile and adjust any results that contradict it. The SEP is canonical; the generation steps produce physical texture.

### Ac - Access

Verify that the system's orbital infrastructure, customs presence, and approach conditions match the Ac value. High Ac (3-5) should produce visible signs: derelict lanes, hostile patrols, minimal docking, mandatory escorts. Low Ac (0-1) should produce open, well-serviced lanes.

### Hz - Hazard

Check that the dominant planet types and stellar conditions justify the Hz value. Extreme Hz (4-5) should make inhabited bodies feel exceptional, requiring visible survival infrastructure on every settlement.

### Rx - Resource

Confirm that at least one body or installation clearly produces or controls the dominant resource type. If the rolled content does not support Rx, add an installation or adjust a station's function.

### Pp - Population

Verify that the overall population density across the system feels consistent with Pp. Too many inhabited bodies for a low Pp system: reduce. A dense Pp 5 system that feels empty: add stations or increase settlement scale.

### Pw - Power

Ensure that whoever controls the system is physically visible: headquarters, military presence, administrative stations, corporate insignia, or a conspicuous absence of authority if Pw is A or V.

### Tn - Tension

Place active tension markers. High Tn (3-5) should mean contested installations, blockades, scarred infrastructure, or armed patrols. Tn 0 should feel settled and quiet.

### Dx - Distinctiveness

Confirm that the Dx value has a physical home in the system. Dx = X needs an alien biosphere. Dx = H needs ruins or a contested historical site. Dx = T needs an artifact or active research facility. If nothing in the generated content expresses Dx, add it now.


## Step 10: Local conditions

Every detailed system should end with a summary of local operating conditions.

Record:

- Travel safety.
- Prohibited zones.
- System-wide hazards.
- Likely encounter types.
- Active routes (drawn from the network pass — see the main generation procedure).
- Important rumors or verifiable facts.
- A short referee-facing summary.


## Referee guidance

Do not feel obliged to make every system equally complex. Some systems should be simple and functional. Others should become major locations with multiple worlds, several stations, and a strong identity.

The SEP is the stable core. The details are the visible body.

## Optional rule: density of detail

Use the SEP to decide how much detail to generate.

- Low Pp systems may only need one major body and a few installations.
- Moderate Pp systems should usually include multiple worlds and at least one station or habitat.
- High Pp systems should include layered settlement, transit infrastructure, and administrative complexity.
- High Dx systems should receive extra detail, even if population is low.

This keeps the system manageable while allowing important locations to stand out.

## Closing principle

A system is not only a point on a map. It is a structure of bodies, habitats, stations, settlements, control, and meaning.

The SEP gives you the structure in compact form. This expansion turns that structure into a place.
